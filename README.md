# DEALS-STORE

DEALS-STORE is a small full-stack deals and wishlist web application. The
current directory now contains a container-ready setup with a static frontend,
a PHP REST API, and a MySQL database.

Users can browse discounted offers, create an account, sign in, and save or
remove favorite deals from a personal wishlist.

## What Changed In This Directory

- Added a Docker Compose setup for the full app stack:
  - `frontend` runs Nginx and serves the static HTML/CSS/JavaScript files on
    port `80`.
  - `backend` runs the PHP API on port `8000`.
  - `mysql-db` runs MySQL on port `3306` with a persistent Docker volume.
- Updated the backend database host from `127.0.0.1` to `mysql-db` so the PHP
  container can connect to the MySQL container over the Compose network.
- Added a MySQL healthcheck so the backend waits for the database before
  starting.
- Added separate Docker networks:
  - `frontend-net` connects the frontend and backend.
  - `backend-net` connects the backend and database.
- Updated frontend HTML asset paths from `/frontend/css/...` and
  `/frontend/js/...` to `css/...` and `js/...` so the files work correctly when
  served from the Nginx document root.
- Added backend Docker build support with `backend/Dockerfile` and
  `backend/.dockerignore`.
- Kept local runtime files such as `.env` and `workbench/` ignored by Git.

## Main Features

- Browse current deals on the Discover page.
- Register a new user account.
- Sign in and keep the active user in browser local storage.
- Save deals to a personal wishlist.
- Remove saved deals from the wishlist.
- Seed the database with sample offers.

## Tech Stack

- Frontend: HTML, CSS, vanilla JavaScript
- Frontend runtime: Nginx
- Backend: PHP 8.3 CLI with PDO MySQL
- Database: MySQL
- Local app orchestration: Docker Compose

## Database

The database schema is defined in `database/schema.sql`.

It creates these tables:

- `users`: stores account details and password hashes.
- `offers`: stores deal information, prices, discounts, images, and expiry
  dates.
- `favorites`: connects users to saved offers.

The `offers.final_price` column is generated by MySQL from `old_price` and
`discount_percentage`.

## API Endpoints

The frontend expects the backend API to be available at:

```text
http://localhost:8000/api
```

Available endpoints:

| Method | Endpoint | Description |
| --- | --- | --- |
| `POST` | `/api/auth.php` | Register or log in a user. Uses an `action` value of `register` or `login`. |
| `GET` | `/api/offers.php` | Returns all offers ordered by newest first. |
| `GET` | `/api/favorites.php?user_id=ID` | Returns a user's saved offers. |
| `POST` | `/api/favorites.php` | Adds or removes a favorite. Uses an `action` value of `add` or `remove`. |

## Running With Docker Compose

Start the full stack from the project root:

```bash
docker compose up -d
```

On the first database startup, Docker mounts `database/schema.sql` into MySQL so
the required tables are created automatically.

Open the frontend:

```text
http://localhost/
```

The backend API is exposed at:

```text
http://localhost:8000/api
```

The MySQL database is exposed on:

```text
localhost:3306
```

Stop the stack:

```bash
docker compose down
```

Remove the database volume if you need a fresh database:

```bash
docker compose down -v
```

## Seeding Sample Offers

After the containers are running, insert demo offers from inside the backend
container:

```bash
docker compose exec backend php seed.php
```

You can also test the database connection from the backend container:

```bash
docker compose exec backend php config/test_db.php
```

## Manual Development Notes

The current backend database host is `mysql-db`, which is correct inside Docker
Compose. If you run the PHP backend directly on your host machine instead of in
the backend container, update `backend/config/db.php` or provide an equivalent
host mapping so PHP can reach MySQL.

Manual backend command:

```bash
php -S localhost:8000 -t backend
```

Manual frontend command:

```bash
php -S localhost:8080 -t frontend/html
```

## Important Files

- `docker-compose.yml`: defines the frontend, backend, database, volumes,
  healthcheck, and networks.
- `backend/Dockerfile`: builds the PHP API container with PDO MySQL enabled.
- `backend/config/db.php`: PDO database connection settings.
- `backend/api/auth.php`: register and login endpoint.
- `backend/api/offers.php`: offer listing endpoint.
- `backend/api/favorites.php`: wishlist add, remove, and list endpoint.
- `backend/models/User.php`: user registration and login logic.
- `backend/models/offer.php`: offer query logic.
- `backend/models/Favorite.php`: wishlist query logic.
- `database/schema.sql`: MySQL database schema.
- `frontend/html/index.html`: main deals listing page.
- `frontend/html/login.html`: sign-in page.
- `frontend/html/register.html`: account creation page.
- `frontend/html/favourit.html`: wishlist page.
- `frontend/js/app.js`: shared frontend logic for loading offers, auth UI,
  login, logout, and favorites.
- `frontend/css/style.css`: shared visual design.

## Notes

- The backend allows cross-origin requests so the frontend can call the API
  during local development.
- Database credentials are currently defined in `docker-compose.yml` and
  `backend/config/db.php`; keep them in sync if you change them.
- `.env`, `.gitignore`, Dockerfiles named `Dockerfile`, and `workbench/` are
  ignored by the current `.gitignore` rules.

# DEALS-STORE

DEALS-STORE is a simple full-stack deals and wishlist web application. It uses a static HTML/CSS/JavaScript frontend, a PHP backend API, and a MySQL database.

The app lets users browse discounted offers, create an account, sign in, and save or remove favorite deals from a wishlist.

## Project Structure

```text
DEALS-STORE/
├── backend/
│   ├── api/              # PHP API endpoints for auth, offers, and favorites
│   ├── config/           # Database connection and connection test script
│   ├── models/           # PHP classes for users, offers, and favorites
│   └── seed.php          # Inserts sample offers into the database
├── database/
│   └── schema.sql        # MySQL schema for users, offers, and favorites
├── frontend/
│   ├── css/              # App styling
│   ├── html/             # Browser pages
│   └── js/               # Frontend app logic and API calls
├── docker-compose.yml    # MySQL service configuration
└── workbench/            # Local workbench/runtime files
```

## Main Features

- Browse current deals on the Discover page.
- Register a new user account.
- Sign in and store the active user in browser local storage.
- Save deals to a personal wishlist.
- Remove saved deals from the wishlist.
- Seed the database with sample offers.

## Tech Stack

- Frontend: HTML, CSS, vanilla JavaScript
- Backend: PHP with PDO
- Database: MySQL
- Local database runtime: Docker Compose

## Database

The database schema is defined in `database/schema.sql`.

It creates these tables:

- `users`: stores account details and password hashes.
- `offers`: stores deal information, prices, discounts, images, and expiry dates.
- `favorites`: connects users to saved offers.

The `offers.final_price` column is generated by MySQL from `old_price` and `discount_percentage`.

## API Endpoints

The frontend expects the backend API to be available at:

```text
http://localhost:8000/api
```

Available endpoints:

| Method | Endpoint | Description |
| --- | --- | --- |
| `POST` | `/api/auth.php` | Register or log in a user. Uses an `action` value of `register` or `login`. |
| `GET` | `/api/offers.php` | Returns all offers ordered by newest first. |
| `GET` | `/api/favorites.php?user_id=ID` | Returns a user's saved offers. |
| `POST` | `/api/favorites.php` | Adds or removes a favorite. Uses an `action` value of `add` or `remove`. |

## Running Locally

### 1. Start MySQL

```bash
docker compose up -d mysql-db
```

On the first database startup, Docker mounts `database/schema.sql` into MySQL so the required tables are created automatically.

### 2. Seed Sample Offers

After MySQL is running, insert demo offers:

```bash
php backend/seed.php
```

You can also test the database connection with:

```bash
php backend/config/test_db.php
```

### 3. Start the Backend API

Run the PHP backend on port `8000`:

```bash
php -S localhost:8000 -t backend
```

This makes the API available under `http://localhost:8000/api`.

### 4. Start the Frontend

From the project root, serve the static frontend on another port:

```bash
php -S localhost:8080 -t .
```

Then open:

```text
http://localhost:8080/frontend/html/index.html
```

## Important Files

- `frontend/html/index.html`: main deals listing page.
- `frontend/html/login.html`: sign-in page.
- `frontend/html/register.html`: account creation page.
- `frontend/html/favourit.html`: wishlist page.
- `frontend/js/app.js`: shared frontend logic for loading offers, auth UI, login, logout, and favorites.
- `frontend/css/style.css`: shared visual design.
- `backend/config/db.php`: PDO database connection settings.
- `backend/models/User.php`: user registration and login logic.
- `backend/models/offer.php`: offer query logic.
- `backend/models/Favorite.php`: wishlist query logic.

## Notes

- The backend currently allows cross-origin requests so the frontend can be served separately during development.
- Database connection values are defined in both `docker-compose.yml` and `backend/config/db.php`; keep them in sync if you change them.
- `workbench/`, `.env`, and local runtime files are ignored by Git.
